# Claude Task List — Seed Content

## One-liner

Claude Code tasks, unchained from sessions. See, launch, and resume — right from your menu bar.

## Origin Story

I was using Claude Code's built-in task list system daily — not just for coding tasks, but for everything. Groceries, errands, project planning. It was faster and more natural than any external tool connected via MCP.

But there was friction: task lists are bound to sessions. Every time I opened a new terminal, I had to run a command to load my lists. I wanted my tasks to just *be there* — always visible, always ready.

So I built Claude Task List: a lightweight macOS menu bar app that watches `~/.claude/tasks/` and gives you a simple, instant view of everything — with one-click execution.

## The Problem

Claude Code power users manage their entire workflow through Claude — but task lists disappear when the session ends. Current alternatives:

- **`/tasklist:load` every time** — works, but tedious repetition
- **Linear/Notion via MCP** — tasks get created, but they're just text. No session context, no execution link. You're copy-pasting fragments of context into a dumb external system.

## What Makes This Different

### 1. Tasks carry their session context
When you spawn a task, Claude already has the full context from when the task was created. No briefing needed. No context loss. External tools can only store task *text* — this stores the task's *brain*.

### 2. Claude is both creator and executor
Tasks aren't imported from outside — Claude creates them while working, and Claude executes them. The task system and the execution engine are one and the same. It's the most organic integration possible.

### 3. Cross-list, cross-session natural adds
You're coding and suddenly remember you need to pick up mail? Just tell Claude to add it to your errands list. It goes to the right place, from any session. No context switching, no app switching.

### 4. Simple by design
Menu bar. Click. See everything. Execute. That's it. No dashboards, no config screens, no onboarding. It does one thing and does it instantly.

## Target User

People who:
- Use Claude Code as their **primary workflow engine**, all day
- Want Claude Code as their **Single Source of Truth** — the hub where everything connects
- Are automating their entire workflow through Claude — coding, planning, personal tasks, everything
- Live in the terminal and want *less* context switching, not more

## Vision

Today: a task viewer and executor.

Tomorrow: **a launchpad for your personal agent army.**

- Each task list becomes an **agent initiating bed** — where Ralph loops are kicked off
- Persistent Claude sessions connected via Telegram channels become always-on **ghosts** working for you
- The menu bar becomes the **command center** — see what your agents are doing, what tasks they're processing, initiate new work
- Think: OpenClaw-style autonomous agent orchestration, but from your menu bar

**From task list to launch pad. From one Claude session to a fleet.**
